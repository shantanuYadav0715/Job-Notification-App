01

Design Principles

P-01
Clarity Before Completeness

Every element on screen must earn its place. When in doubt, remove it. Information that cannot be immediately understood is not useful information — it is noise. Present only what the user needs at this moment in their workflow.

P-02
Structure Creates Calm

Consistent spacing, predictable layout patterns, and a defined visual hierarchy allow users to build mental models quickly. Uncertainty in layout produces cognitive load. Certainty in layout produces confidence.

P-03
State Is Always Visible

The system must reflect its own state without ambiguity. Loading, empty, error, and success states are first-class citizens — not afterthoughts. Users should never wonder what is happening or what has happened.

P-04
Precision Over Decoration

Visual details exist to communicate, not to impress. Typography conveys hierarchy. Color conveys meaning. Spacing conveys relationship. Nothing is decorative. Every visual decision has a functional rationale.

P-05
Tone Earns Trust

This product handles something personal — a person's professional livelihood. Language, error messages, and microcopy must be composed, direct, and respectful. The system never blames, never overpromises, and never trivializes.

P-06
Interaction Has Reason

Motion and transition serve orientation. They communicate change, confirm action, and reduce disorientation — never attract attention to themselves. If removing an animation produces a better experience, remove it.

P-07
Consistency Is the Product

Users do not evaluate individual screens — they form impressions of a system. Consistent radius, consistent spacing, consistent tone, and consistent behavior compound into a product that feels considered and reliable.

When NOT to Use Accent (#8B0000)

Do Not Use For

Large background fills or full-bleed sections. Text longer than a label or link. Decorative dividers or borders with no semantic meaning. Secondary or tertiary actions — accent is reserved for primary intent only. Error states — accent is not an error color.

4-Color Rule

At any given moment in the UI, no more than four colors should be visually present: Background, Text, one Surface tone, and one semantic color (Accent, Success, or Warning). This constraint forces hierarchy and prevents visual fragmentation.

04

Spacing System

Five values only. No other values are permitted. All padding, margin, and gap must resolve to one of these values.

--spacing-8
8px — Inline gaps, label-to-input, icon-to-label, badge padding, tight internal rhythm.
--spacing-16
16px — Component internal padding (small), list item rhythm, form field stacking, nav items.
--spacing-24
24px — Card padding, modal padding, content group spacing, medium component padding.
--spacing-40
40px — Section spacing within a panel, major content breaks, between related groups.
--spacing-64
64px — Page-level section separation, full-bleed horizontal padding, top/bottom page breathing room.

Vertical Rhythm Rule: All vertical measurements resolve to multiples of 8. Never use odd values (e.g., 12px, 20px, 36px). Whitespace communicates separation — generous space between sections signals a structural break; tight space within a component signals a relationship.

07

Interaction Principles

Motion Philosophy

Motion communicates state change, not personality. Every transition must answer: what just changed, and why should the user notice? Duration is 150ms for instant feedback (hover, focus). 200ms for confirmations and reveals. No easing that overshoots. No parallax. No sequence animations between unrelated elements.

Feedback Hierarchy

Tier 1: Inline visual confirmation (border change, icon change). Tier 2: Toast / banner for system-level outcomes. Tier 3: Full-page state for critical transitions. Use the lowest tier that adequately communicates the change. Never escalate feedback without cause.

Error Messaging Tone

State what happened. Explain why it happened. Offer the specific action to fix it. Never use passive voice. Never say "something went wrong." Never blame the user. Example: "We couldn't save your preferences because the session timed out. Save again to continue."

Empty State Philosophy

An empty state is a first-use state. It must orient the user, state what this area is for, and offer the precise action that will populate it. Never show a blank container. Never show only an illustration. Always include one actionable sentence.

Micro-interaction Rules

Hover: border or background shift. 150ms. Focus: visible outline — never remove for sighted keyboard users. Loading: inline skeleton placeholder in same layout position. Never full-page spinner for partial content loads. Disabled: cursor:not-allowed. Opacity 0.5 on the label only, not the container.

Transition Spec

All transitions use ease-in-out. Fast: 150ms. Base: 200ms. No spring. No bounce. No delay chains. Properties to transition: color, background-color, border-color, opacity. Do not transition layout properties (width, height, margin) in the main flow.
